# Learning Paths: Your Roadmap

Choose a path based on your timeline and goals.

---

## Quick Assessment

| Timeline | Best Path |
|----------|-----------|
| < 1 week | Sprint Path |
| 1-4 weeks | Focused Path |
| 1-3 months | Comprehensive Path |
| 3+ months | Mastery Path |

---

## Sprint Path (1 Week)

**For:** Last-minute preparation, refreshers

### Day 1-2: Core Patterns
- [ ] Two Pointers
- [ ] Sliding Window
- [ ] Binary Search
- [ ] BFS/DFS

### Day 3-4: Data Structures
- [ ] Arrays & Strings problems
- [ ] Hash Table problems
- [ ] Tree problems
- [ ] Graph problems

### Day 5: System Design
- [ ] Framework (RESHADED)
- [ ] 2 case studies (URL Shortener, Twitter)

### Day 6: Behavioral
- [ ] STAR method
- [ ] Prepare 5 stories
- [ ] Practice out loud

### Day 7: Mock & Review
- [ ] Full mock interview
- [ ] Review weak areas
- [ ] Rest before interview

## Focused Path (1 Month)

**For:** Targeted preparation with some flexibility

### Week 1: Foundations
- [ ] Big-O notation
- [ ] All data structures review
- [ ] 20 easy problems

### Week 2: Core Patterns
- [ ] Two Pointers, Sliding Window
- [ ] Binary Search
- [ ] BFS/DFS, Trees
- [ ] 30 medium problems

### Week 3: Advanced Patterns + System Design
- [ ] Dynamic Programming basics
- [ ] Backtracking
- [ ] System Design fundamentals
- [ ] 3 case studies

### Week 4: Polish
- [ ] Behavioral prep (10 stories)
- [ ] Mock interviews (2-3)
- [ ] Company-specific prep
- [ ] Weak area review

---

## Comprehensive Path (3 Months)

**For:** Career changers, thorough preparation

### Month 1: Build Foundation

**Week 1-2: Data Structures**
- [ ] Arrays, Strings, Linked Lists
- [ ] Stacks, Queues, Hash Tables
- [ ] Trees, Heaps, Graphs
- [ ] 50 easy problems

**Week 3-4: Core Patterns**
- [ ] Two Pointers, Sliding Window
- [ ] Binary Search
- [ ] BFS/DFS
- [ ] 40 medium problems

### Month 2: Deepen Skills

**Week 5-6: Advanced Patterns**
- [ ] Dynamic Programming (all patterns)
- [ ] Backtracking
- [ ] Greedy algorithms
- [ ] 50 medium problems

**Week 7-8: System Design**
- [ ] Fundamentals & building blocks
- [ ] 5 case studies
- [ ] Practice designing systems

### Month 3: Polish & Specialize

**Week 9-10: Behavioral & LLD**
- [ ] STAR stories (15+)
- [ ] LLD problems (3-5)
- [ ] Mock behavioral interviews

**Week 11-12: Final Prep**
- [ ] Company-specific preparation
- [ ] Full mock interviews (4-5)
- [ ] Review and strengthen weak areas

## By Target Company

### FAANG/Big Tech
```
Focus areas:
- Coding: Medium-Hard LeetCode (150+ problems)
- System Design: 8-10 case studies
- Behavioral: 15+ STAR stories
- Timeline: 2-3 months recommended
```

### Startups
```
Focus areas:
- Coding: Easy-Medium (80+ problems)
- System Design: Practical, not theoretical
- Take-home projects: Practice 2-3
- Timeline: 1 month sufficient
```

### Finance/Trading
```
Focus areas:
- Coding: Medium-Hard, emphasis on optimization
- Math/probability questions
- System Design: Low latency focus
- Timeline: 2 months recommended
```

## Daily Schedule Template

### Weekday (2-3 hours)

```
Morning (1 hour):
- 2 coding problems

Evening (1-2 hours):
- 1 system design topic OR
- Pattern deep dive OR
- Behavioral prep
```

### Weekend (4-6 hours)

```
Session 1 (2-3 hours):
- 4-5 coding problems
- Focus on weak areas

Session 2 (2-3 hours):
- System design case study
- Mock interview practice
```

---

## Key Takeaways

1. **Consistency > intensity.** 2 hours daily beats 10 hours weekends.
2. **Pattern recognition** is the goal, not problem count.
3. **Mock interviews** are non-negotiable.
4. **Rest before interviews.** Fresh mind > extra cramming.
