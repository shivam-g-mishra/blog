# Netflix & Uber Interview Guides

Two companies known for strong engineering cultures but very different approaches.

---

## Netflix

### Culture: Freedom & Responsibility

```
Netflix Culture Memo highlights:
- "Adequate performance gets a generous severance"
- Freedom with accountability
- Context, not control
- Highly aligned, loosely coupled
```

### What They Look For

| Trait | Example |
|-------|---------|
| **Judgment** | Make wise decisions with incomplete info |
| **Communication** | Candid, direct feedback |
| **Impact** | Accomplish amazing amounts |
| **Curiosity** | Learn rapidly and eagerly |
| **Innovation** | Re-conceptualize issues |
| **Courage** | Say what you think |
| **Passion** | Inspire others |
| **Selflessness** | Seek what's best for Netflix |

### Interview Process

```
1. Recruiter Screen
2. Technical Phone Screen (coding)
3. Onsite (4-6 rounds)
   - Coding × 2
   - System Design × 1-2
   - Behavioral/Culture × 2
```

### Key Tips

- **Be direct.** Netflix values candor.
- **Show judgment.** Decisions with ambiguity.
- **Demonstrate impact.** Quantified achievements.
- **Know their tech.** Microservices, streaming, Java.

---

## Uber

### Culture: Fast-Paced Execution

```
Uber values:
- Move fast
- Celebrate cities
- Be an owner
- Persist through challenges
```

### Interview Process

```
1. Recruiter Screen
2. Phone Screen (1-2 coding rounds)
3. Onsite (5-6 rounds)
   - Coding × 2-3
   - System Design × 1-2
   - Behavioral × 1
```

### Focus Areas

| Team | Key Skills |
|------|------------|
| **Rider/Driver** | Mobile, real-time systems |
| **Maps/Routing** | Algorithms, optimization |
| **Eats** | Marketplace dynamics |
| **Platform** | Distributed systems |
| **Payments** | Financial systems, consistency |

### System Design Topics

```
Likely topics:
- Design Uber (ride matching)
- Design surge pricing
- Design real-time location tracking
- Design Uber Eats delivery optimization
- Design payment splitting
```

### Key Tips

- **Speed matters.** Uber values quick iteration.
- **Think scale.** Millions of trips daily.
- **Real-time focus.** Low latency requirements.
- **Multi-sided marketplace.** Balance rider/driver needs.

## Comparison

| Aspect | Netflix | Uber |
|--------|---------|------|
| **Culture** | Freedom + Responsibility | Move Fast |
| **Tech Stack** | Java, microservices | Go, Java, Python |
| **Interview Style** | Culture-heavy | Technical-heavy |
| **System Design** | Streaming, CDN | Real-time, geo |
| **Pay** | Top of market | Competitive |

---

## Key Takeaways

### Netflix
1. **Culture fit is critical.** Read the culture memo.
2. **Be candid.** Direct communication valued.
3. **Show judgment.** Decisions with incomplete info.

### Uber
1. **Think real-time.** Everything is live.
2. **Scale mindset.** Millions of concurrent users.
3. **Move fast.** Iteration speed matters.
