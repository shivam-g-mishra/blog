# Microsoft Interview Guide

Microsoft's interview is more balanced than pure algorithm-focused companies. They value problem-solving, collaboration, and growth mindset.

---

## Interview Process

```
1. Recruiter Screen (30 min)
   └── Background, motivation, team fit

2. Phone Screen (45-60 min)
   └── 1-2 coding problems

3. Onsite / Virtual Loop (4-5 rounds)
   ├── Coding × 2
   ├── System Design × 1 (Senior+)
   ├── Behavioral × 1 ("As Appropriate")
   └── Hiring Manager × 1
```

| Value | What They Look For |
|-------|-------------------|

| **Customer Obsession** | Focus on customer impact |

## Coding Expectations

### Focus Areas
- Data Structures (arrays, trees, graphs)
- Algorithms (sorting, searching, DP)
- String manipulation
- Object-oriented design

### Expectations by Level

| Level | Difficulty | Notes |
|-------|------------|-------|
| SDE | Easy-Medium | Clear communication valued |
| SDE II | Medium | Optimal solutions expected |
| Senior | Medium-Hard | Design thinking + coding |
| Principal | Medium | Focus on architecture |

---

## The "As Appropriate" Round

Microsoft's version of behavioral. Tests culture fit and growth mindset.

### Common Questions

- "Tell me about a time you failed"
- "How do you handle disagreements?"
- "Describe learning something new"
- "Tell me about a project you're proud of"

### What They're Looking For

```
Growth Mindset indicators:
✓ Learning from failures
✓ Embracing challenges
✓ Seeking feedback
✓ Helping others grow

Red flags:
✗ Blaming others
✗ Fixed mindset ("I'm just not good at X")
✗ Not acknowledging mistakes
```

## System Design (Senior+)

### Expectations
- Clear requirement gathering
- Scalable architecture
- Trade-off discussions
- Azure services awareness (helpful, not required)

### Common Questions
- Design a file storage system
- Design a search engine
- Design a real-time collaboration tool
- Design a CI/CD pipeline

## Key Takeaways

1. **Growth mindset is real.** Show you learn from failures.
2. **More balanced** than pure algorithm tests.
3. **Collaboration matters.** Microsoft values teamwork.
4. **Azure knowledge helps** but isn't required.
5. **Talk through your thinking.** Communication is valued.
