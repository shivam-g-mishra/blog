# Amazon Interview Guide

Amazon's interview is defined by Leadership Principles. You'll hear them in every round.

---

## Interview Process

```
1. Recruiter Screen (30 min)
   └── Background, LP questions

2. Online Assessment (Optional)
   └── 2 coding problems + work simulation

3. Phone Screen (45-60 min)
   └── Coding + LP questions

4. Onsite Loop (4-5 rounds)
   ├── Coding × 2
   ├── System Design × 1 (SDE2+)
   └── Leadership Principles × 2 (mixed in all rounds)
```

---

## The 16 Leadership Principles

### Most Asked

| Principle | Focus |
|-----------|-------|
| **Customer Obsession** | Put customer first |
| **Ownership** | Act like an owner |
| **Invent and Simplify** | Find simpler solutions |
| **Dive Deep** | Stay connected to details |
| **Deliver Results** | Focus on outcomes |
| **Have Backbone, Disagree and Commit** | Respectfully challenge |

### Also Important

| Principle | Focus |
|-----------|-------|
| Earn Trust | Listen, speak candidly |
| Learn and Be Curious | Never stop learning |
| Hire and Develop the Best | Raise the bar |
| Insist on Highest Standards | Never "good enough" |
| Bias for Action | Speed matters |
| Frugality | Do more with less |

---

## Behavioral Format

**Every question maps to a Leadership Principle.**

```
Interviewer: "Tell me about a time you had to make a decision 
without complete information."

[Bias for Action, Ownership, or Deliver Results]

Use STAR:
- Situation: Context and stakes
- Task: Your responsibility
- Action: What YOU did (specific steps)
- Result: Outcome with metrics if possible
```

## Coding Expectations

### By Level

| Level | Difficulty | Expectations |
|-------|------------|--------------|
| SDE I | Easy-Medium | Solve 1-2 problems, clean code |
| SDE II | Medium | Solve optimally, discuss trade-offs |
| SDE III | Medium-Hard | Optimal solution, system thinking |

### Focus Areas
- Arrays and Strings
- Trees and Graphs
- Hash Maps
- Dynamic Programming basics

---

## System Design (SDE2+)

### What They Look For
- **Scale estimation:** Back-of-envelope math
- **Trade-off discussions:** Always discuss alternatives
- **Dive deep:** Be ready to go into any component

### Common Questions
- Design a recommendation system
- Design an e-commerce order system
- Design a notification system
- Design Prime Video streaming

## Prepare Your Stories

**Have 8-10 STAR stories ready:**

| Story Type | Covers Principles |
|------------|-------------------|
| Led a project | Ownership, Deliver Results |
| Fixed a customer issue | Customer Obsession |
| Simplified something complex | Invent and Simplify |
| Went into details to find issue | Dive Deep |
| Disagreed with someone | Have Backbone |
| Learned something new | Learn and Be Curious |
| Made decision with incomplete info | Bias for Action |
| Did more with less | Frugality |

---

## The Bar Raiser

One interviewer is a "Bar Raiser" from another team:
- Independent voice
- Ensures hiring bar
- Can veto hire
- Usually asks deeper behavioral questions

---

## Key Takeaways

1. **Leadership Principles dominate.** Prepare STAR stories for each.
2. **Customer Obsession is #1.** Always tie back to customer impact.
3. **Ownership means accountability.** "I" not "we."
4. **Data-driven results.** Include metrics when possible.
5. **Bar Raiser has veto power.** Every interviewer matters.
